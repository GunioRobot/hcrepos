\section{Hello UNIX!!}
私が大学生の頃の情報処理の授業ではSolarisという大型計算機に
たくさんのディスプレイをつないで
\footnote{大嘘。細かい話をすると異なる仕組み。}使っていました。
このSolarisというのが，情報系の学生ならば一度くらいは耳にしたことがあるだろう，{\bf UNIX}
\footnote{「ゆにっくす」と読むが，ある界隈では「うにっくす」とも読む．}に分類されるシステムなのです．

私がUNIX\footnote{UNIX is not Linux.}とHelloしたのもこのときである。
私が大学時代につまづきそうになったところなどを中心に以下ではUNIXを
とりまくツールの使い方について説明する。

\section{Hello emacs!!}
おそらく大部分の人が最初に当たる壁が{\bf emacs}だろう。
emacsとはUNIXで広く使われているエディタである．
{\bf メモ帳}\footnote{Windows付属のエディタ，Notepad.exeのこと}とは
比較にならないくらい高機能，高性能なもので、
emacsはプログラミングして拡張できるという特徴から
エディタではないものとして扱われることもあるほどだ。

以下では，emacsの使い方について簡単に説明しておく．

\paragraph{基本操作}
まず，emacsで基本的な操作はすべて
{\bf Ctrlを押しながら他のキーを押す}ことにより行う．
はじめはこの感覚に戸惑うかもしれないが，
慣れればかなり快適
に操作することができる．
ちなみに私はCarbonCocoaというMac OS Xで動作するEmacsを使ってこの文書を修正・加筆している．

このEmacsの操作方法を説明するときに良く\verb|C-?|という表現を用いる．
これはCtrlキーを押しながら何かのキーを押すという操作を示している。
同様にMetaキーを押しながらの操作は\verb|M-?|
\footnote{多くの環境ではAltがMeta Keyの動作をします．}
と表現する．
たとえば，Ctrl + X であれば，\verb|C-x|というように表現される．
ここでも広く使われているこの表記にしたがって説明する。

\subsection{起動}
起動は\verb|emacs kogaidan.txt|といったコマンドを入力して行う．
ここで，kogaidan.txtは開きたいテキストファイルを指定する(なければ保存時に作成される．)．

\subsection{終了}
終了は\verb|C-x C-c|という操作で行う．
この説明でまだ慣れない方も多いと思うので簡便な表現で言い換えると、
Ctrlを押しながらX,Cという順番に押せばよい．

ファイルに変更があった場合は保存するかを聞かれる．
保存するのであれば"y"を，保存しないのであれば"n"を入力する。

\subsection{保存}
ファイルの保存は \verb|C-x C-s| という操作で行う．

名前をつけてファイルを保存したいときは \verb|C-x C-w| と操作する．
その後，ファイル名を聞かれるので入力すればよい．
これを覚えておけば，起動時にファイル名を入力し忘れても、
あとからファイルに名前をつけて保存することができる。

\subsection{他の操作}
\subsubsection{移動，削除}
Ctrlとの組み合わせで行うこともできるが，
入門ではカーソルによる移動やバックスペースによる操作くらいで十分だろう．

\subsubsection{コピーや切り取り}
慣れない場合、マウスが使える環境であればこれを利用することをすすめる。
Emacsに慣れた人間はリージョンという概念を用いてコピーや切り取りを行う。

\subsubsection{移動が不便}
HomeやEndといったキーを優雅に使いこなすWindowsユーザにとっては
Emacsでの移動が不便なものに感じられるかもしれない．
しかし，それは大きな誤解でEmacsにはEmacsの移動方法がある。

試しに\verb|C-a|\footnote{行頭へ移動}や
\verb|C-e|\footnote{行末へ移動}
を押すとWindowsのHomeやEndに相当する移動ができることが確認できる。

\subsubsection{変な操作をしてしまったとき}
たとえば，保存や終了をする気もないのに
\verb|C-x|を押してしまったときにはどうすればよいのだろうか？

答えは簡単で，{\bf emacsでのキャンセル動作は C-g を連打}と考える．
おかしなカスタマイズなどをしていない場合はこれで入力待ち状態に戻れる．


\section{Hello C!!}
UNIXのプログラミング言語といえばC言語を使う．
大規模教育環境でのプログラミングといえば、
多くの人にとってはじめてのプログラミングになる可能性があった。

正直なところ，{\bf C言語(の入門)は死ぬほど退屈}である．
おそらく，情報処理の授業で扱う程度のプログラミングでは，
結果が文字で表示されるというだけのものしか作れないだろう．
その退屈な手間を低減する方法を以下で紹介する．

\begin{enumerate}
\item emacs -nw

プログラミングにはemacsを使うことになるだろう．
そこで，便利な使い方を紹介しよう．
emacsを\verb|emacs -nw kogaidan.c|
\footnote{kogaidan.cはC言語のプログラムを記述したファイル}
として起動することで、コマンドを入力したウィンドウで開くことができる．

\item Ctrl + Z

これではターミナルがEmacsに奪われてしまい、
コマンドを入力することもできないではないか、と思われるかもしれない。
だが、ターミナルでEmacsを使うことにより、別のメリットが生まれる。
\verb|C-z|を押すことによって，emacsの表示は消え，
再びターミナルにコマンドを入力することができるようになる．
つまり，この状態で\verb|gcc kogaidan.c|のようなコマンドを実行できるのである．

\item fg

再びemacsの画面に戻るには\verb|fg|というコマンドを
入力してあげればよい．

以降はこの繰り返しで編集，コンパイル，実行を
繰り返すことができる．

ここで気づいたが
\verb|emacs -nw|である必要はあまりないかもしれない．
筆者の好みなのである．

\end{enumerate}
